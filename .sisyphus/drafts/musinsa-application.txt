프로젝트 1: log8 - AI와 함께 운영하는 개인 기술 블로그

기간: 2024.12 - 현재 (운영 중)
역할: 1인 개발 및 운영
기술 스택: Astro v5, TailwindCSS v4, TypeScript, Cloudflare Pages, Bun
배포 URL: https://log8.kr
GitHub: https://github.com/IISweetHeartII/My_Website_Astro

[프로젝트 배경과 목표]

"개발자에게 글쓰기가 왜 중요한가?"라는 질문에서 시작된 프로젝트입니다. 저는 학습한 내용을 정리하고, 문제 해결 과정을 기록하며, 이를 다른 개발자들과 공유하고 싶었습니다. 기존의 Tistory, Velog 같은 플랫폼도 좋지만, 저만의 브랜딩과 완전한 커스터마이징이 가능한 개인 블로그를 갖고 싶었습니다.

[AI와 함께하는 개발 방식]

이 프로젝트에서 가장 특별한 점은 AI를 '도구'가 아닌 '페어 프로그래밍 파트너'로 활용했다는 것입니다.

저는 최근 급부상한 OpenCode와 Oh My OpenCode(ohmyopencode) 플러그인을 적극적으로 활용하고 있습니다. 이 플러그인은 "Sisyphus"라는 이름의 오케스트레이션 에이전트를 제공하는데, 그리스 신화의 시시포스처럼 반복적이지만 끈기 있게 작업을 수행합니다. 22만 스타를 받은 이 프로젝트를 보자마자 "이거다!" 싶어서 바로 실행에 옮겼습니다.

[구체적인 AI 활용 사례]

1. 코드베이스 분석과 리팩토링
   - 기존 컴포넌트 구조가 복잡해지면서 유지보수가 어려워졌을 때, Claude Code에 "Feature-First Architecture로 리팩토링 제안해줘"라고 요청
   - AI가 기존 코드를 분석하고, features/, shared/ 구조로 마이그레이션 계획을 수립
   - 단순히 제안만 받는 게 아니라, 제가 설계 의도를 설명하면 AI가 더 정교한 구조를 제안하는 방식으로 협업

2. TailwindCSS v4 마이그레이션
   - TailwindCSS v3에서 v4로 업그레이드할 때, 문법 변경 사항(예: bg-gradient-to-* → bg-linear-to-*)을 AI가 자동으로 감지하고 수정 제안
   - 수십 개의 파일에서 일관된 패턴으로 수정해야 했는데, AI와 함께 AST-grep을 활용해 안전하게 일괄 변경

3. SEO/AEO 최적화
   - "AI 시대에 블로그가 검색엔진과 AI 어시스턴트 모두에게 잘 노출되려면?"이라는 질문으로 시작
   - robots.txt에 GPTBot, ClaudeBot, PerplexityBot, Applebot-Extended 등 AI 크롤러를 명시적으로 허용
   - 각 포스트마다 Schema.org 마크업을 적용해 구조화된 데이터 제공

[기술적 도전과 해결]

문제 1: Obsidian과의 원활한 연동
- Obsidian에서 글을 쓰면 바로 블로그에 반영되는 워크플로우가 필요
- src/content/blog/ 폴더를 Obsidian vault로 설정, publish: true frontmatter로 게시 여부 제어
- Astro Content Collections API를 활용해 빌드 타임에 자동으로 필터링

문제 2: 한국어 타이포그래피 최적화
- Pretendard 폰트 적용으로 한글 가독성 향상
- font-display: swap으로 FOUT 방지
- Lighthouse 성능 점수 90+ 달성

문제 3: CI/CD 자동화
- GitHub Actions + Cloudflare Pages 연동
- PR 생성 시 자동 프리뷰 배포
- Biome를 활용한 린팅/포맷팅 자동화

[성과와 배운 점]

- 40개 이상의 기술 포스트 작성 및 운영
- Lighthouse 성능 90+ 달성
- Google AdSense 승인 획득
- AI와의 협업을 통해 혼자서는 생각하지 못했을 아키텍처 개선 아이디어를 얻음
- "완벽한 코드"보다 "실행과 개선"의 사이클이 중요하다는 것을 체득

프로젝트 2: 덕친소 - AI 기반 소개팅 매칭 서비스

기간: 2025.11 - 현재 (3개월, 운영 중)
역할: 풀스택 1인 개발
기술 스택: Next.js, Spring Boot, MySQL, Docker Compose, Mac Mini M2 서버
배포: Cloudflare Tunnel (Zero Trust), Vercel

[프로젝트 배경]

"김덕환 친구 소개" - 이름 그대로입니다. 주변 친구들로부터 "나 소개팅 좀 해줘"라는 요청이 끊이지 않았습니다. 처음에는 기억에 의존해서 매칭해줬지만, 인원이 늘어나면서 한계에 부딪혔습니다.

"이걸 시스템화하면 어떨까?"라는 생각에서 시작해, 결국 AI 기반 자동 추천 시스템으로 발전시켰습니다.

[AI 활용 - 문제 해결의 핵심]

1단계: 수동 매칭의 한계
- 친구 A의 이상형: "유머 있는 사람, 키 175 이상"
- 친구 B의 특징: "ENFP, 183cm, 개그맨 지망생"
- 이런 정보를 엑셀로 관리하다가 공수가 기하급수적으로 증가

2단계: 벡터 DB + AI 추천으로 전환
- 이상형 데이터와 친구 특징(키, MBTI, 성격, 취미 등)을 벡터 DB로 저장
- 유사도 기반 매칭 알고리즘 구현
- AI가 "왜 이 사람을 추천했는지" 설명까지 생성

3단계: AI 코딩 어시스턴트와의 협업
- Mac Mini M2에 백엔드를 배포하는 과정에서 Claude Code를 적극 활용
- Cloudflare Tunnel 설정, Docker Compose 구성, SSL 인증서 자동 갱신 등
- "이런 에러가 나는데 어떻게 해결해야 해?"라고 물으면 AI가 로그를 분석하고 해결책 제시
- 혼자였으면 며칠 걸렸을 작업을 몇 시간 만에 완료

[기술적 도전과 해결]

문제 1: 개인 서버 배포의 복잡성
- 집에 있는 Mac Mini M2를 서버로 활용하고 싶었지만, 포트포워딩과 보안 문제가 복잡
- 해결: Cloudflare Tunnel(Zero Trust)을 활용해 외부 노출 없이 안전하게 서비스 배포
- AI가 Cloudflare 공식 문서를 참조하며 단계별 설정 가이드 제공

문제 2: 알림 시스템 구축
- 처음에는 카카오 알림톡을 연동하려 했으나 사업자 등록 없이는 불가능
- 해결: Solapi를 활용한 SMS 알림으로 대체
- 매칭이 성사되면 양쪽에게 자동으로 문자 발송

문제 3: 결제 시스템 통합
- 향후 프리미엄 서비스 확장을 위해 결제 시스템 연동 필요
- 포트원(PortOne) API를 활용한 결제 플로우 구현
- 테스트 결제까지 완료, 실제 서비스 적용 준비 완료

[성과와 비전]

- 현재 실제 서비스 운영 중, 지인들 대상으로 매칭 진행
- 향후 프리미엄 결혼정보회사 형태로 확장 계획
- AI를 활용한 "설명 가능한 매칭"의 가능성 검증

프로젝트 3: Finders - 필름 현상소 연결 플랫폼

기간: 2025.12 - 현재 (2개월, 진행 중)
역할: Backend Lead
기술 스택: Java 21, Spring Boot 3.4, MySQL 8.x, Spring Data JPA, QueryDSL, GCP
GitHub: https://github.com/Finders-Official/BE

[프로젝트 배경]

"뷰파인더 너머, 취향을 찾다" - 필름 카메라 사용자들이 현상소를 찾고, 예약하고, 사진을 공유할 수 있는 플랫폼입니다. UMC 9기에서 11명의 팀원(백엔드 5, 프론트 4, PM 1, 디자인 1)과 함께 진행하고 있습니다.

[나의 역할과 AI 활용]

BE Lead로서의 책임:
- 전체 백엔드 아키텍처 설계 및 코드 리뷰
- GCP 인프라 구축 (Compute Engine, Cloud Storage, Cloud SQL)
- CI/CD 파이프라인 설정 및 배포 자동화
- 팀 내 코드 스타일 가이드 및 컨벤션 문서화

AI와 함께한 핵심 기능 개발:

1. Stable Diffusion Inpainting API 연동 - 필름 빛번짐 복원
   - 필름 사진의 빛번짐(Light Leak)을 AI로 복원하는 기능
   - Stable Diffusion API 연동 과정에서 Claude Code 활용
   - 프롬프트 엔지니어링을 통해 필름 특유의 질감은 유지하면서 빛번짐만 제거

2. GCP 클라우드 인프라 설계 및 VPC 네트워크 구축
   - finders-vpc 커스텀 VPC 생성, 서브넷 설계 (10.0.0.0/20 대역)
   - Cloud SQL과 Compute Engine 간 VPC 피어링 설정으로 프라이빗 IP 통신 구현
   - IAP(Identity-Aware Proxy) 기반 SSH 터널링으로 외부 포트 노출 없이 보안 접속
   - 팀원들이 로컬에서 DB 접속할 수 있도록 SSH 터널 스크립트 작성 및 공유

3. 포트원(PortOne) 결제 시스템 연동
   - 현상소 예약 결제 플로우 설계 (예약 → 결제 → 확정)
   - Webhook 기반 결제 상태 동기화 구현
   - 테스트/프로덕션 환경 분리 및 결제 검증 로직 구현

4. Virtual Threads 도입 (Java 21)
   - 동시성 처리 성능 향상을 위해 Java 21의 Virtual Threads 적용
   - AI가 기존 ThreadPool 코드를 Virtual Threads로 마이그레이션하는 방법 제안

[기술적 도전과 해결]

문제 1: 보안과 편의성을 모두 잡는 클라우드 인프라 설계
- Cloud SQL에 외부 IP 없이 접속해야 하는 보안 요구사항
- 해결: VPC 피어링 + IAP SSH 터널링 조합으로 해결
- GCE와 Cloud SQL을 같은 VPC 내 프라이빗 IP로 연결 (외부 노출 제로)
- 개발자들은 IAP 인증 후 SSH 터널을 통해 localhost:3307로 DB 접속
- 팀 공유용 db-tunnel.sh 스크립트 작성으로 접속 편의성 확보

문제 2: 팀 코드 품질 관리
- 5명의 백엔드 개발자가 일관된 코드 스타일 유지 필요
- 해결: 상세한 CODE_STYLE.md, CONVENTIONS.md 문서 작성
- PR마다 빡빡한 코드 리뷰 진행, KPT 회고 주도

[성과]

- 현재 v0.4.2 릴리즈 (555 commits)
- 체계적인 도메인별 아키텍처 설계 (domain/, infra/, global/)
- OpenAPI(Swagger) 기반 API 문서화 자동화

프로젝트 4: 오메추 - AI 기반 메뉴 추천 서비스

기간: 2025.06 - 2025.08 (3개월)
역할: Frontend Developer (인증 파트)
기술 스택: Next.js, TypeScript, Zustand, TanStack Query, Vercel
배포 URL: https://omechu.log8.kr
GitHub: https://github.com/Team-Omechu/Omechu-web
수상: UMC 8기 데모데이 장려상, 현장 투표 1위

[프로젝트 배경]

"오늘 메뉴 뭐 먹을까?" - 매일 반복되는 고민을 AI로 해결하는 서비스입니다. 사용자의 취향과 위치 정보를 고려해 최적의 메뉴를 추천합니다.

[경계를 넘는 문제 해결]

프론트엔드 담당이었지만, 팀 전체의 성공을 위해 백엔드 영역까지 파고들었습니다.

문제 상황:
- 원인 불명의 인증 오류로 팀 전체 개발이 2주간 중단
- CORS 에러, 401 Unauthorized가 랜덤하게 발생
- 백엔드 팀도 원인을 찾지 못하는 상황

나의 행동:
1. "내 업무가 아니다"라고 선을 긋지 않고, 직접 백엔드 로그 분석 시작
2. 쿠키/세션 기반 로그인과 JWT 기반 카카오 로그인이 혼재되어 있음을 발견
3. 문제 흐름을 블로그로 정리하며 논리적으로 분석
4. JWT 기반 통합 인증 방식을 제안하고 팀을 설득

결과:
- API 응답 속도 420ms → 210ms로 50% 개선
- Stateless한 JWT의 특성을 활용해 서버 부하 감소
- 팀의 개발 속도 정상화

[AI 활용]

- 인증 시스템 통합 과정에서 Claude에게 "JWT와 세션 방식의 차이점, 각각의 장단점"을 질문
- Access/Refresh 토큰 관리 best practice 학습
- 카카오 소셜 로그인 + JWT 통합 아키텍처 설계

[데모데이 이후 - FSD 아키텍처 마이그레이션]

데모데이 수상 후에도 프로젝트 품질 개선을 멈추지 않았습니다.

- 기존 폴더 구조의 한계를 느끼고 Feature-Sliced Design(FSD) 아키텍처로 마이그레이션 제안
- AI를 활용해 수십 번 코드베이스를 분석하며 최적의 구조 설계
- 내 담당 도메인뿐 아니라 팀원들의 코드까지 적극적으로 리뷰
- app/, entities/, features/, shared/, widgets/ 레이어로 관심사 분리
- 마이그레이션 후 새 기능 추가 시 개발 속도 체감상 2배 향상

[성과]

- UMC 8기 데모데이 장려상 수상
- 현장 투표 1위 달성
- 인증 시스템에 대한 깊은 이해 획득
- FSD 아키텍처 실전 적용 경험

프로젝트 5: Starterm - 스타트업 단어 학습 플랫폼

기간: 2025.08 - 2025.09 (3주)
역할: Full-Stack Developer (3명 팀)
기술 스택: Next.js, Supabase, TailwindCSS, Framer Motion, Vercel
배포 URL: https://starterm.co.kr

[프로젝트 배경]

한양대학교 창업학 수업에서 "예창패, MVP 같은 스타트업 용어를 처음 접하는 학생들이 어려움을 겪는다"는 문제를 발견하고, 3주 만에 MVP를 개발해 실제 사용자를 확보한 프로젝트입니다.

[3주 만에 MVP 배포 - AI의 역할]

린 개발의 핵심 - AI로 백엔드 시간 단축:
- Supabase를 선택한 이유: "3주 안에 인증, DB, 실시간 기능을 모두 구현해야 한다"
- Claude에게 "Supabase로 인증 + 실시간 기능을 빠르게 구현하는 방법"을 질문
- 백엔드 구축 시간을 대폭 단축하고, 핵심 기능 개발에 집중

Framer Motion으로 인터랙티브 UX:

- 카드 플립, 진도 바 애니메이션 등
- AI가 Framer Motion 공식 문서 기반으로 최적화된 코드 제안

- 3주 만에 기획부터 배포까지 완료

왜 저인가? - AI Native Engineer로서의 정체성

저는 AI를 "도구"로 쓰는 것을 넘어, AI와 함께 일하는 것이 자연스러운 개발자입니다.

1. 새로운 AI 도구가 나오면 즉시 실험합니다.
   - OpenCode 오케스트레이션 기능(Oh my open code)을 보자마자 바로 실행
   - Claude Code, Cursor, Antigravity 등 다양한 AI 코딩 도구 경험
   - 단순 사용을 넘어, 각 도구의 장단점을 파악하고 상황에 맞게 선택

2. AI와의 협업으로 불가능을 가능으로 바꿉니다.
   - 덕친소: 혼자서 Mac Mini 서버 배포까지 완료
   - log8: Feature-First Architecture로 리팩토링
   - Finders: Stable Diffusion Inpainting 연동

3. 배움을 나누는 것을 아끼지 않습니다.
   - 시니어 소상공인 대상 AI 활용 강연 3회 진행 (한국시니어교류협회 1회, 리피움 2회)
   - ChatGPT Custom Instructions, Gemini Gems, NotebookLM, MCP 활용법 소개
   - 새로운 것을 배우면 팀원과 지인에게 공유하는 것을 시간 아끼지 않고 실천

저는 '아끼지 않는 사람'입니다.
- 돈을? 새로운 것을 도전하고 경험하는 데 투자합니다.
- 시간을? 배운 것을 공유하고 나누는 데 아끼지 않습니다.

무신사 ROOKIE AI NATIVE ENGINEER로서, AI와 함께 무신사의 기술 혁신에 기여하고 싶습니다.
